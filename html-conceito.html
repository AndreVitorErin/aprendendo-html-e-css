HTML - Conceitos

Agora que já conhecemos a origem da linguagem HTML, é hora de conhecer melhor seus fundamentos, conceitos e aplicações.

A primeira coisa que convem destacar é que HTML não é uma linguagem de programção, mas sim uma linguagem de marcação, ela estrutura o conteúdo. Toda a parte de programção que ocorre numa pagina web (interatividade) vem do JavaScript. Um documento HTML é escrito de forma declarativa: a gente não precisa explicar o como deve ser feito, isso o navegador resolve pra gente.

A idéia do HTML é compor elementos, que por sua vez, são determinados por tags. E essas tags podem conter atributos que alteram o comportamento desse elemento. As maioria das tags precisam de abertura e fechamento, e todas tags que agem dessa forma são dependentes do seu conteúdo. Em tags que funcionam por abertura e fechamamento, o conteúdo fica entre as tags e será alterado pelas suas definições.

Exemplo de tag com abertura de fechamento: tag de parágrafo (<p>). Que funciona da seguinte maneira:
<p>conteúdo que será modificado por essa tag</p>

Quando uma tag não tem conteúdo (ou seja, tem apenas atributos) ela não é fechada. Como exemplo, temos a tag de imagem (<img>). Que funciona da seguinte forma:
<img src="endereço.da.imagem" alt="texto.alternativo.que.aparecerá.caso.a.imagem.não.carregue">
Pode conter outros atributos, dependendo da necessidade.

Todos os atributos devem conter valores.

A importância do HTML é semântica, ou seja, de significado, de descrever o conteúdo. Portanto toda parte de estilo e formatação deve ser delegada às CSS. No início a linguagem HTML era responsável por essas duas instâncias, mas isso acabou comprometendo o desenvolvimento web, e decidiu-se por separar essas duas instâncias. Ainda existem tags remanecentes que se preocupam exclusivamente com questão de estilo, mas elas não devem ser usadas. Por exemplo, antigamente existia a tag <b> que deixava o texto em negrito. Hoje se usa a tag <strong>, embora o efeito visual possa ser o mesmo, o foco da tag strong é informar ao navegador que aquele(s) termos(s) possuem destaque dentro do texto, o que pode facilitar em mecanismos de buscas.

Para ter acesso a uma documentação mais detalhada, diretamente do site da mozilla, basta clicar aqui.

Outro fator que faz ser relvante a êfase semântica na HTML são os leitores utilizados por deficientes visuais. Esses leitores que transliteram seu site em forma de áudio lêem HTML. Por isso é importante que toda parte de estilo seja feita via CSS, para evitar complicações nessa transposição.

Principais tags HTML de marcação:

  • <p> → parágrafo
  • <h1> a <h6> ;→ títulos
  • <span> → trechos a serem destacados
  • <div> → divisão da página
  • <img> → imagem
  • <table> <tr> <td> → definição de tabela, linhas e colunas
  • <ul> <li> → lista, itens da lista
  • <strong> → destaque (efeito visual, negrito)
  • <em> → ênfase (efeito visual, itálico)
  • <a> → âncoras (links)

Os títulos têm ordem de importância, o que também influencia em dispositivos de busca. H1 é o título principal, a recomendação é que haja apenas um por página. H6 é o título de menor relevância, os outros são uma escala entre estes.

Span é uma tag não-visual, mas que destaca o texto, e posteriormentes você consegue aplicar CSS com mais facilidade no trecho destacado.

A div é semelhante ao span, mas ela é um elemento conteiner, dentro dela você tem outras tags, enquanto a span só tem o conteúdo textual.

A tag ul representa listas não-ordenadas, para listas ordenadas usamos a tag <ol>

Caminho absoluto x Caminho Relativo

O caminho absoluto deve ser evitado, pois é definido o local absoluto em que o recurso (arquivo, imagem, etc) se encontra em disco , por isso, em geral, só funciona no ambiente original do site/página.

Já o caminho relativo define o local relativo do recurso, ou seja, onde esse recurso está em relação aos demais arquivos do projeto. Utilizando sempre o caminho relativo, as referências ao seu projeto funcionarão em qualquer ambiente.

Exemplo de caminho absoluto: C:\Users\Joao\Documents\Projeto-site

Exemplo de caminho relativo: imagens/internetantigamente.jpeg(nesse casim dentro da pasta em que o index do seu site se localiza, existe uma outra pasta chamada "imagens" e dentro dela está o arquivo solicitado)
